# User Story Standard

Defines the standard user story format.

## Format
- **Actor:** Who is the user?
- **Goal:** What are they trying to achieve?
- **Business Value:** Why is it valuable?

## Quality Rules
User stories express intent rather than implementation.

**Examples of Acceptable Stories:**
- As an administrator, I want to view audit logs so that I can monitor compliance.

**Examples of Unacceptable Stories:**
- As a user, I want a database table created so that I can store data.
