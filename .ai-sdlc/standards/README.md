# AI SDLC Enterprise Engineering Standards

This directory contains the permanent engineering standards governing software architecture, coding practices, documentation, repository organization, review processes, security, testing, naming conventions, Git workflows, and folder structures.

## Governing Authority
All engineering standards are strictly subordinate to the AI SDLC Constitution:
[.ai-sdlc/constitution/AI-SDLC-v1.0.md](../constitution/AI-SDLC-v1.0.md)

## Purpose and Hierarchy
These standards act as the authoritative reference for every AI worker and every future software project built using the AI SDLC.

The structure is broken down by discipline to allow independent versioning:
- `architecture/` - Architecture Standard
- `coding/` - Coding Standard, Naming Standard
- `documentation/` - Documentation Standard
- `repository/` - Repository Standard, Git Standard, Folder Standard
- `review/` - Review Standard
- `security/` - Security Standard
- `testing/` - Testing Standard

## Rule Format and Classifications
Every standard follows a strict rule format. Every rule is explicitly classified as:
- **Mandatory:** Must be followed. No exceptions unless explicitly authorized by the Chief Architect.
- **Recommended:** Highly advised, but may be circumvented with valid technical justification.
- **Optional:** Provided as a guideline, but not enforced.

**AI workers must never treat recommendations or optional rules as mandatory requirements.**
